## ■ セクション 1.1：HTMLってなに？

### 1. HTMLの役割

Webサイトの画面に表示されている文字や画像、ボタンなどは、すべて**HTML**(エイチ・ティー・エム・エル)という言語で書かれています。岡山理科大学のWebサイトもHTMLが使われています。

<img height="500" alt="OUS_web" src="https://github.com/user-attachments/assets/48f8d990-2a14-4e6e-8b49-14809d7b929d" />

HTMLの役割は、Webサイトの骨組み（文章の構造）」を作ることです。人間でいうと「骨」、家でいうと「柱や壁」を作る作業にあたります。

### 2. 基本の書き方：タグで「挟む」

HTMLは、テキストを専用の記号で挟むことで、「ここからここまでが見出し」「ここは文章」とコンピューターに教えてあげます。この記号を「タグ」と呼びます。

- **開始タグ**：要素の始まりを表します（例：`<span>`）
- **終了タグ**：要素の終わりを表します。必ずスラッシュ `/` を入れます（例：`</span>`）

```html
<開始タグ>画面に表示したい文字</終了タグ>

<!-- 以下のような形で表されます -->
<p>こんにちは！</p>
<span>こんにちは！</span>

<!-- タグの中にタグが入ることもあります -->
 <p><span>こんにちは！</span></p>
```

### 3. 初心者が絶対に守るべき「超・鉄則」

プログラミングの世界では、たった1文字の間違いで画面が真っ白になったり、デザインが崩れたりします。以下のルールを最初から体に叩き込みましょう。

- **必ず「半角英数字」で書くこと！**
- タグのアルファベットはもちろん、`<` や `>`、`/` もすべて半角です。
- 一番よくある失敗は「全角スペース」が紛れ込んでしまうことです。全角スペースがあると、コンピューターがコードを正しく読めなくなります。日本語を入力した後は、必ずキーボードを半角入力に戻す癖をつけましょう。

### ✍️ 【ミニワーク】最初のファイルを作って、文字を表示させてみよう！

1. `01_chapter1`のフォルダを指定します。
2. 右クリックして、新しいファイルをクリックします。
3. 新しいファイルを作成し、名前を **`index.html`** にします。

- _※Webサイトのトップページとなるファイルは、必ず `index.html` という名前にするルールになっています。_

4. ファイルの中に `Hello World!` とだけ書いて、ファイルを保存（Ctrl + S または Cmd + S）します。

## ■ セクション 1.2：これだけ知っていればOKな「5大タグ」

Webサイトで使われているタグは何種類もありますが、実は**よく使う5つのタグを覚えるだけで、世の中のWebサイトの9割は作れてしまいます。**
自己紹介制作課題でも大活躍する「5大タグ」をマスターしましょう！

### ① `<div>`（ディブ）

- **意味**：特別な意味を持たない、透明な「グループ化のための箱」。
- **使い方**：HTMLの中で一番よく使う超重要タグです。「ヘッダーの塊」「自己紹介の塊」など、複数の要素をひとまとめにしたいときに使います。

### ② `<h1>` 〜 `<h6>`（エイチワン 〜 エイチシックス）

- **意味**：見出し（Heading）を表します。
- **使い方**：`<h1>` が一番大きな見出しで、数字が大きくなるにつれて小さな見出しになります（一般的には `<h3>` くらいまでをよく使います）。

### ③ `<p>`（ピー）

- **意味**：文章・段落（Paragraph）を表します。
- **使い方**：ニュースの本文や、自己紹介のテキストなど、普通の文章を書くときはすべてこのタグで挟みます。

### ④ `<a>`（エー）

- **意味**：他のページやサイトへ飛ばす「リンク（Anchor）」を作ります。
- **使い方**：これまでのタグとは違い、どこに飛ばすかというURLの情報（**属性**と呼びます）を書き足す必要があります。

```html
<a href="https://example.com">ここをクリックすると飛びます</a>
```

### ⑤ `<img>`（イメージ）

- **意味**：画面に「画像（Image）」を表示します。
- **使い方**：このタグは中に文字を挟まないため、**終了タグ（`</img>`）がありません。** 代わりに、画像の場所を指定する `src`（ソース）属性と、画像の説明を書く `alt`（オルト）属性をセットで書きます。

```html
<img src="my-photo.jpg" alt="私の顔写真" />
```

## ■ セクション 1.3：【最重要】「箱の入れ子構造（親子関係）」

これからWeb制作を学ぶ上で、**一番大切で、かつ一番差がつく概念**です。

### 1. Webサイトは「四角い箱の集まり」

普段見ているWebサイトは綺麗なデザインをしていますが、HTMLの視点で見ると、実は「大小さまざまな四角い箱が、パズルのように組み合わさっているだけ」です。

課題を例に見てみましょう。

<img width="721" height="876" alt="Image" src="https://github.com/user-attachments/assets/6f4b977c-6c15-4745-b9e7-e3cca793ae15" />

- 一番外側に「サイト全体」という大きな箱があります。
- その中に「ヘッダーの箱（赤色）」「ヘッダー画像（青色）」「Aboutの箱（緑色）」「Likesの箱（オレンジ色）」「フッターの箱（紫色）」が縦に並んでいます。
- 「Likesの箱」の中には、さらに「趣味カード1」「趣味カード2」「趣味カード3」という小さな箱が横に並んでいます。

このように、箱の中にさらに箱を入れる構造を「入れ子構造」**、または**「親子関係」と呼びます（外側の箱を「親要素」、内側の箱を「子要素」と言います）。

### 2. コードを読みやすくする「インデント（Tabキー）」の絶対ルール

箱の中に箱を入れるとき、そのまま縦に書き並べると、どれが親でどれが子なのか分からなくなってしまいます。
そこで、**子要素（内側の箱）を書くときは、行の頭にスペースを空ける**というルールがあります。これを**インデント**と呼びます。

- キーボードの **`Tab` キー** を1回押すと、綺麗なスペースが空きます。
- VS Codeなどのエディタでは、Enterで改行すると自動でインデントを入れてくれることが多いです。

❌ **悪い例（インデントがない・バラバラ）**
どこで箱が始まって、どこで閉じているのか天才でも分かりません。エラーの原因になります。

<img width="1672" height="171" alt="Image" src="https://github.com/user-attachments/assets/3f59a424-cf48-4461-8f42-773d3f9f7864" />

⭕ **良い例（インデントが綺麗に揃っている）**
`<div>` という親の箱の中に、`<h2>` と `<p>` という子要素が入っていることが一目で分かります。

```html
<div>
  <h2>趣味</h2>
  <p>映画鑑賞</p>
</div>
```

### ⚠️ よくある初心者エラーと対策

「画面が崩れた！」「文字が表示されない！」という先輩たちの失敗の9割は、この第1章のルールが守れていないことが原因です。動かないときは以下をチェックしましょう。

1. **タグの閉じ忘れ、閉じ順の間違い**

- 親の箱を閉じる前に、子要素の箱を閉じなければいけません。挟む順番がごちゃ混ぜにならないように注意しましょう。

2. **全角の英数字や全角スペースが混ざっている**

- 特に日本語を入力した直後は要注意です。
